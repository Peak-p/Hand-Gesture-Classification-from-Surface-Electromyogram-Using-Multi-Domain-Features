การจำแนกท่าทางของมือด้วยสัญญาณคลื่นไฟฟ้ากล้ามเนื้อที่ผิวหนังโดยใช้คุณลักษณะโดเมนที่หลากหลาย

ข้อมูลสำหรับการแยกแยะท่าทางมือโดยใช้สัญญาณไฟฟ้ากล้ามเนื้อ (sEMG) โปรเจกต์นี้เราดึงฟีเจอร์หลายๆ แบบ (Multi-domain) มาใช้คู่กับโมเดล Machine Learning แบบ Ensemble เพื่อให้ทายแม่นขึ้น ทั้งในคนปกติและผู้สูญเสียอวัยวะ

ภาพรวม
การรู้จำท่าทางมือให้แม่นๆ สำคัญมากสำหรับพวกแขนเทียมและการสั่งงานด้วยคอมพิวเตอร์ เราเลยเสนอวิธีดึงข้อมูลแบบจัดเต็มถึง 456 ฟีเจอร์ต่อ 1 ช่วงเวลา (Segment) ซึ่งประกอบไปด้วย:

- Time-domain (เช่น RMS, Zero Crossings, Waveform Length)

- Frequency-domain (เช่น Median Frequency, Band Power)

- Autoregressive (AR)

- Hjorth (Mobility, Complexity)

- จากนั้นเอาฟีเจอร์พวกนี้ไปทดสอบกับโมเดล XGBoost, Random Forest และ LightGBM ทั้งแบบทดสอบในคนเดียวกัน (Intra) และทดสอบข้ามคน (Inter)

  ชุดข้อมูลที่ใช้ (Dataset)
- ชุดข้อมูล Ninapro DB10 (MeganePro) จาก https://ninapro.hevs.ch/instructions/DB10.html

- มีคนปกติ 30 คน + ผู้สูญเสียอวัยวะ 15 คน

- บันทึกด้วยเครื่อง Delsys Trigno Wireless แบบ 12 ช่องสัญญาณ

- เก็บข้อมูลที่ความถี่ 1,926 Hz

- มีท่าทางมือทั้งหมด 10 แบบ

  วิธีทำ (Methods)
- การเตรียมข้อมูล: แบ่งข้อมูลแบบ Sliding window (ขนาด 500 ms, 1000 ms และ 2000 ms) ขยับทีละ 200 ms

- การสกัดฟีเจอร์: ได้ 38 ฟีเจอร์ต่อช่องสัญญาณ × 12 ช่อง = 456 ฟีเจอร์ต่อ 1 Window

- โมเดลที่ใช้: XGBoost, Random Forest, LightGBM

  ผลลัพธ์ (Results)
ความแม่นยำตอนทดสอบในคนเดียวกัน (Intra-participant):

  - คนปกติ: แม่นสูงสุด 83% (ใช้ LightGBM คู่กับ Window ขนาด 2000 ms)

  - ผู้สูญเสียอวัยวะ: แม่นสูงสุด 71% (ใช้ LightGBM คู่กับ Window ขนาด 2000 ms)

ความแม่นยำตอนเทสต์ข้ามคน (Inter-participant): 
  - ค่อนข้างต่ำ (คนปกติ 25–29%, ผู้สูญเสียอวัยวะ 13–17%) ทำให้เห็นชัดเจนว่าสัญญาณกล้ามเนื้อของแต่ละคนมีความต่างกันมากจริงๆ

ข้อค้นพบสำคัญ (Key Insights)
- ยิ่งใช้ Window ใหญ่ขึ้น โมเดลยิ่งแม่น แต่ข้อเสียคือจะประมวลผลช้าลง (หน่วงขึ้น)

- การใช้ฟีเจอร์หลายๆ แบบรวมกัน ช่วยให้ดึงเอกลักษณ์ของสัญญาณ sEMG ออกมาได้ดีและครอบคลุม

- การปรับจูนโมเดลให้เข้ากับเฉพาะบุคคล (Subject-specific calibration) ยังเป็นเรื่องที่ขาดไม่ได้ โดยเฉพาะกับผู้สูญเสียอวัยวะ

สิ่งที่จะทำต่อในอนาคต (Future Work)
- ลองใช้อัลกอริทึมที่ปรับตัวได้เก่งขึ้น (เช่น Transfer learning หรือ Semi-supervised learning)

- ลองเอาเซนเซอร์แบบอื่นๆ มารวมด้วย (เช่น เซนเซอร์รับการสัมผัส + เซนเซอร์วัดความร้อน)
